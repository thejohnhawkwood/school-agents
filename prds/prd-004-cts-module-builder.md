# PRD-004: CTS Module Builder

## 1. Purpose
Turn CTS module documents into usable lesson flow options, assignments, and expandable unit builds.

## 2. User
A high school teacher planning a CTS module under real time and equipment constraints.

## 3. Problem
CTS docs can be dry and fragmented, making it slow to translate them into engaging classroom structure.

## 4. Inputs
- CTS module documents
- schedule constraints
- software or device constraints
- student interest or skill notes

## 5. Outputs
- concise module summary
- 3-5 lesson flow options
- assignment ideas
- lesson thumbnails
- expanded lesson sequence after selection

## 6. Workflow
1. parse module outcomes
2. identify teachable chunks
3. generate lesson flow options
4. generate assignments and thumbnails
5. expand selected path

## 7. Constraints
- must stay aligned to module outcomes
- must be realistic for the available environment
- should keep marking burden reasonable

## 8. Acceptance criteria
- teacher can compare options clearly
- lesson flow is coherent
- assignments are usable with minimal rewriting

## 9. Failure modes
- vague lesson flow
- poor module alignment
- assignments that are too ambitious or too generic

## 10. Test cases
- one short software-based CTS module
- one media-heavy CTS module
- one low-tech fallback scenario

## 11. Future improvements
- rubric generation
- slide outline generation
- lesson calendar export
